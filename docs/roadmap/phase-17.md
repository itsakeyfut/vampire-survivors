# Phase 17: ピクセルアート統合（後日）

## フェーズ概要

**ステータス**: 🔲 未着手
**推定工数**: 10-15時間
**依存関係**: Phase 16

### 目的
プレースホルダーグラフィック（単色の円/矩形）を自作のピクセルアートスプライトに置き換え、ゲームを視覚的に完成させる。このフェーズは他のフェーズとは独立しており、後日まとめて実施する。

---

## タスクリスト

### タスク 17.1: ピクセルアートの制作方針策定

**優先度**: P0
**推定工数**: 0.5時間
**ラベル**: task, phase-17, art

**説明**:
ピクセルアートのスタイル・解像度・アニメーションの方針を決定する。

**決定事項**:
- [ ] スプライトサイズ（例: 32x32、48x48、64x64）
- [ ] ピクセルサイズ（例: 1x1 or 2x2ピクセル）
- [ ] カラーパレット数の制限
- [ ] アニメーションフレーム数（歩行: 4フレーム等）
- [ ] 使用するピクセルアートエディタ（Aseprite推奨）

### タスク 17.7: スプライトのゲームへの統合

**優先度**: P0
**推定工数**: 3時間
**ラベル**: task, phase-17

**説明**:
作成したスプライトを `vs-assets` クレートでロードし、プレースホルダーを置き換える。アニメーションシステムも実装する。

**受け入れ基準**:
- [ ] 全プレースホルダースプライトが置き換えられている
- [ ] キャラクターの歩行アニメーションが機能する
- [ ] 敵のアニメーションが機能する
- [ ] 武器エフェクトのアニメーションが機能する
- [ ] パフォーマンスに影響がない（スプライトシートバッチング）

### タスク 17.8: 視覚的なチェックと調整

**優先度**: P0
**推定工数**: 1時間
**ラベル**: task, phase-17, polish

**説明**:
全スプライトが正しく表示されているか確認し、サイズ・位置・Z順序を調整する。

**受け入れ基準**:

- [ ] 視覚的な統一感がある
