# Vampire Survivors クローン - 実装ロードマップ

## 概要

このディレクトリには、Vampire Survivorsクローンプロジェクトの実装ロードマップが含まれています。各フェーズは独立したロードマップファイルとして管理され、GitHub Issueとして起票できるレベルの詳細なタスクリストを含んでいます。

## ロードマップの使い方

1. **現在のフェーズを確認**: 下記の進行状況表を参照
2. **フェーズのロードマップを開く**: 該当する phase-XX.md を開く
3. **タスクをIssue化**: 各タスクをGitHub Issueとして起票
4. **実装**: タスクの受け入れ基準を満たすまで実装
5. **完了チェック**: 各タスクの完了条件を確認
6. **次のフェーズへ**: フェーズの全タスク完了後、次フェーズに進む

## 凡例

- ✅ **完了**: フェーズの全タスクが完了し、検証済み
- 🚀 **進行中**: フェーズのタスクを実装中
- 🔲 **未着手**: まだ開始していない
- ⏸️ **保留**: 何らかの理由で一時停止中
- ❌ **ブロック**: 依存関係により開始できない

## マイルストーン

### M1: MVP（Minimum Viable Product）
**目標**: 基本的なゲームループが動作する状態

- Phase 1〜6 完了
- プレイヤーが移動し、武器で敵を倒せる
- XPを集めてレベルアップできる
- HPが0でゲームオーバーになる

**完了予定**: Phase 6 完了時

---

### M2: フル機能版
**目標**: ゲームの全機能が実装された状態

- Phase 1〜14 完了
- 全武器・全敵・ボスが実装されている
- 武器進化システムが機能する
- メタ進行（ゴールドショップ）が機能する

**完了予定**: Phase 14 完了時

---

### M3: ポリッシュ版
**目標**: BGM・SFX・エフェクトが整い、遊べる状態

- Phase 1〜16 完了
- BGMと効果音が統合されている
- エフェクトが充実している
- ゲームバランスが調整されている

**完了予定**: Phase 16 完了時

---

### M4: 最終版
**目標**: ピクセルアートが統合された完全版

- Phase 1〜17 完了
- 全プレースホルダーがピクセルアートに置き換えられている
- 視覚的な統一感がある

**完了予定**: Phase 17 完了時

## GitHub Issue起票ガイド

### タスクのIssue化方法

各フェーズのロードマップには、以下の形式でタスクが記載されています：

```markdown
#### タスク X.Y: タスク名

**優先度**: P0/P1/P2/P3
**推定工数**: X-Y時間
**ラベル**: task, phase-X

**説明**:
タスクの詳細説明

**受け入れ基準**:
- [ ] AC1: 条件1
- [ ] AC2: 条件2

**実装ガイド**:
実装の手順や注意点
```

**Issueタイトル:**
```
[Phase X] タスク名
```

**ラベル設定:**
- `task` または `enhancement`
- `phase-X`（例: `phase-2`）
- 優先度: `P0`, `P1`, `P2`, `P3`

### タスクの優先度

- **P0 (Critical)**: ゲームの基本機能に必須。最優先で実装
- **P1 (High)**: 重要な機能。早めに実装すべき
- **P2 (Medium)**: あると良い機能。時間があれば実装
- **P3 (Low)**: 追加機能や改善。後回しでも可

## 進行管理のヒント

### 1. 1フェーズずつ着実に進める
- 複数フェーズを同時並行しない
- 各フェーズの完了条件をすべて満たしてから次に進む

### 2. パフォーマンスを早めに確認する
- Phase 5 完了時に300体同時での60fps確認
- ボトルネックがあれば早期に対処

### 3. 定期的に検証する
- 各タスク完了時に動作確認
- フェーズ完了時に総合テスト

### 4. ゲームバランスのメモを残す
- プレイしながら気になった点を TODO コメントに残す
- Phase 16 のポリッシュフェーズで一括対応

### 5. コミット戦略
- 各タスク完了時にコミット
- コミットメッセージ: `feat(scope): タスク名`
- フェーズ完了時にPRを作成（推奨）
